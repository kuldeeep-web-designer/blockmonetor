🔹 BLOCKCHAIN FUNDAMENTALS
Blockchain is a decentralized and distributed digital ledger that records transactions across multiple computers. Each block contains data, a timestamp, and a cryptographic hash of the previous block.

Key Concepts:
- Decentralization: No central authority controls the network
- Immutability: Data cannot be changed once recorded
- Transparency: All transactions are visible on the network
- Security: Cryptography ensures data integrity

Example:
Bitcoin and Ethereum use blockchain to store transaction data securely.

--------------------------------------------------

🔹 CONSENSUS MECHANISMS
Consensus mechanisms ensure agreement among network participants.

Types:
- Proof of Work (PoW): Requires computational power (used in Bitcoin)
- Proof of Stake (PoS): Validators stake coins (used in Ethereum 2.0)
- Delegated Proof of Stake (DPoS): Voting-based validation

🔹 SMART CONTRACTS (SOLIDITY)
Smart contracts are self-executing programs stored on the blockchain.

Example:
```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.0;

contract SimpleStorage {
    uint256 public value;

    function setValue(uint256 _v) public {
        value = _v;
    }

    function getValue() public view returns (uint256) {
        return value;
    }
}
